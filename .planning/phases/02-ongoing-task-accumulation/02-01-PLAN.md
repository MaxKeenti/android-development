## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User input → file content | NOTES.md and README entries are written by the executor based on task description from the user/teacher |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | NOTES.md content | accept | Personal learning repo, no external readers with trust expectations; content accuracy is self-evident |
| T-02-02 | Information Disclosure | README.md links | accept | Public repo with no secrets; broken links are a usability defect, not a security issue |
| T-02-03 | Tampering | Existing NOTES.md overwrite | mitigate | Task 2 action explicitly creates, not overwrites; executor must check for existing file before writing and stop if the task folder already has a NOTES.md |
</threat_model>
